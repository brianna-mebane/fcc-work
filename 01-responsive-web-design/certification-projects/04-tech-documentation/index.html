Introduction

GitHub is a premier cloud-based platform and tool for developers, teams, and companies alike to store, share, and collaborate on projects in a safe and secure manner. A key feature of GitHub are directories called "repositories" that allow users to showcase their work while also keeping track of the various changes in their projects, viewing suggestions from others on how to improve their projects, and work with collaborators without impacting in the integrity of a project.

GitHub's collaborative working features are made possible thanks to the open-source software Git, a version control system that tracks changes in computer files and creates "snapshots" of a project's code at various points in time for users to refer to in the future for different reasons (e.g. troubleshooting, reverting a project to an older version, etc.).

In order to unlock some of GitHub's most magical powers for your next project, it is imperative that you learn how to access your GitHub repositories from your computer's command line, a text-based interface that allows you to interact with your computer's operating system using lines of text known as "commands". On this page, you will learn how to install GitHub CLI (GitHub's official command line tool) so that you may access your repositories, track your project changes, and make the most of GitHub without having to constantly log on to the platform's website.
